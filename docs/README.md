## Contributing

When adding new documentation:
1. Place files in this `/docs` directory
2. Use Markdown format
3. Update this README with links
4. Include examples and screenshots where helpful
5. Keep technical and user documentation separate

## Documentation Standards

- Use clear, concise language
- Include code examples for technical docs
- Add screenshots for UI features (coming soon)
- Keep docs up-to-date with code changes
- Version documentation when releasing
